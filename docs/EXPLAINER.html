The Link Safety Contract

This laptop shares Ableton Link with other musicians on stage. One stray tempo write would yank their rigs too. So the rig cannot make one — not by policy, by construction:

NEVERwrites tempo — Link tempo is read-only to every line of this code.
NEVERwrites transport — no start, stop, continue, or song-position.
NEVERlaunches scenes — the set's 11 scenes carry baked tempos; firing one would change the room.

Enforced at the bottom of the stack: Resolver.set / Resolver.call are the only paths to the Live API in the whole codebase, and they refuse these unconditionally. There is no bypass — proven by grep on every build. Tempo is read freely (the rig listens to the room); it is never spoken.

the cast

Two brains, one job each

Kept deliberately separate so a failure in one can never take the other down.

device · 40 Master FX

CONDUCTOR

Turns a launched, named clip into a slow, musical gesture — a beat-synced ramp engine that breathes filters, sends and macros over bars, then returns each one to exactly where your hands left it.

  • 15 moves: WASH, TIDE OUT, BREATH, BLOOM, RISE, HORIZON, FOCUS, VEIL, PULSE…
  • Captures before it moves; restores to the captured value, never an assumed default.
  • Grabs a parameter only for the length of a ramp, then lets go.
  • Range sentries skip anything parked outside its safe window — no jumps.
  • CLEAN SLATE / ABORT roll every displaced send back home.
device · Master

SENTINEL

Watches the six bus meters at 10 Hz and trims headroom on the master so a stack of loops can't run the mix into the ceiling — gently, with a soft knee, and only as far as it must.

  • Dominant-bus law over a 4 s attack window; slow release after a comfort window.
  • Per-bus clamps; never more than it needs, evidence logged either way.
  • RITUAL: a one-press pre-show reset & verify (centers filters, zeroes trims, sets capture marks).
  • Hosts the dashboard's telemetry process.
the part that's actually cool

It assumes the stage is hostile

Every one of these is a real failure that happened on the gig laptop, reproduced, and designed out — so the next time it happens, nobody on stage notices.

Someone drags the CONDUCTOR track position

Eduardo moved the CONDUCTOR track to slot 20 and every command went dead.

Resolution was already by-name, but the cached handle pointed at a slot, so observers followed whoever slid into the old slot. Now every track is pinned by object id — it follows the named object anywhere. Move it, rename around it, delete-and-recreate it: by name, it works. No moving it back.

Live restarts and reloads a 104 MB set cold open

The brains wake against an empty Live model; every name misses; a few quick retries die long before the set finishes building.

A retry ladder that spans ~3 minutes, plus on-demand re-resolve from the TEST and RITUAL buttons — recovery is a single button press, never a re-drag.

The moves swept perfectly and moved nothing drive fallback

On this machine, parameter writes sent from the engine's clock silently vanished — the dashboard showed a flawless ramp while the knobs sat still.

A per-write readback watchdog: if a value doesn't land twice running, the rig flips the whole engine to direct API drive and says so, loudly, on the dashboard.

The external clock ran the engine at 1/480th speed trust gate

The wired sync feed delivered beats where the code expected ticks — moves armed but the bar never advanced.

The clock is untrusted by default. External sync only earns trust while it tracks Live's own song position; until it proves itself, a read-only poll of song-time is the authority.

Anything throws, anywhere exception jail

An unhandled exception in a live device can wedge Max — and the show.

Every entry point runs inside a jail: on a throw it releases all grabs, clears state, and logs. Three throws in a minute and the brain self-disables — the set keeps playing, completely untouched, until you re-save the file.

how a press becomes a gesture

The path of a command

Launch a labeled clip on the CONDUCTOR track — that's the entire control surface. No new MIDI mappings.

launch WASH 16 pad→ read clip name→ parse → move + bars→ capture current values→ bar-synced ramp→ grab · drive · release→ restore to captured

Because the command is keyed by the clip's name, you can move the clip to any scene or relabel it to any other valid move — and an unrecognized name simply does nothing, so a mistyped pad never misfires. Full table in the keymap runbook.
